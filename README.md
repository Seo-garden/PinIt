# PinIt

## 1) PinIt 소개

PinIt은 **사진 + 캡션 + 장소**를 하나의 기록으로 저장하고, 이를 지도에서 확인하는 iOS 앱입니다.

## 2) 기능 요구사항 (요약)

- 이메일/비밀번호 기반 인증
- 사진/캡션/장소를 포함한 기록 추가
- 지도에서 기록 확인
- 기록 삭제
- 캡션 수정(자동 저장)

## 3) 상세 요구사항 / 인수조건

### 인증/로그인

- [ ] 사용자는 이메일과 비밀번호를 입력하여 인증할 수 있다.
- [ ] 이메일 입력 시 @ 등 올바른 이메일 형식이 아닐 경우 이메일필드 하단에 “이메일 형식이 올바르지 않습니다” 에러 메시지가 표시되어야 한다.
- [ ] 이메일 형식에 맞는 값이 입력되면 유효성 검증이 통과된 것으로 처리되어야 한다.
- [ ] 비밀번호 필드에 값이 존재할 때 유효성 검증이 통과된 것으로 처리되어야 한다.
- [ ] 모든 필드에 값이 존재하고 유효성 검증이 통과해야만 로그인 버튼이 활성화되어야 한다.
- [ ] 로그인 요청 중에는 로그인 버튼 안에 로딩상태가 표시되어야 한다.
- [ ] 로그인 요청 중에는 로그인 버튼이 비활성화되어야 한다.
- [ ] 로그인 실패 시(등록되지 않은 이메일 또는 비밀번호 불일치) “이메일 또는 비밀번호가 일치하지 않습니다” 메시지를 노출한다.
- [ ] 로그인 성공 시 메인 화면으로 진입한다.

### 기록 추가 폼

- [ ] 사용자는 하나의 사진, 캡션, 하나의 장소로 구성된 기록을 추가할 수 있다.
- [ ] 폼에서 사진 필드는 임의의 기본 풍경 이미지로 설정되어 있어야 한다.
- [ ] 사진 필드 옆 수정 버튼을 누르면 자동으로 카메라 앱으로 전환되어 사용자가 사진을 찍어 업로드할 수 있어야 한다.
- [ ] 이미 선택된 사진은 교체/삭제 할 수 있어야 한다.
- [ ] 폼에서 캡션 필드에는 플레이스홀더(“기록을 표현하는 한 문장을 작성해보세요”)가 존재해야 한다.
- [ ] 캡션 최대 글자 수는 300자(공백 포함) 초과 입력이 제한된다.
- [ ] 폼에서 장소 필드의 위도/경도 값은 기본적으로 비어 있어야 한다.
- [ ] 장소 설정 시 위도/경도 숫자를 입력해서 장소를 설정할 수 있어야 한다.
- [ ] 사진/캡션/장소 중 하나라도 입력되지 않았다면 “기록하기” 버튼이 비활성화되어야 한다.
- [ ] 모든 값이 입력되었을 때 “기록하기” 버튼이 활성화된다.
- [ ] “기록하기” 버튼을 누르면 누른 시간을 “기록 생성 시간”으로 정의하고 기록 데이터에 추가해야 한다.
- [ ] 저장이 완료되면 폼은 화면 상에서 사라져야 한다.
- [ ] 저장 중에는 버튼이 비활성화되고 로딩 표시가 노출되어야 한다.

### 지도/기록 보기

- [ ] 사용자는 지도에 표시된 기록들을 확인할 수 있다.
- [ ] 처음 앱 실행 시 지도의 중심은 사용자의 위치여야 한다. 이때 위치 권한이 없다면 요청한다.
- [ ] 지도의 배율은 100% 여야 한다.
- [ ] 사용자의 위치 권한이 없는 경우 서울 광화문 중심으로 지도를 표시한다.
- [ ] 지도 위의 마커를 클릭했을 때 지도를 반쯤 가리는 모달 형태로 상세 정보(사진 & 장소 & 캡션 & 생성시간)를 확인할 수 있어야 한다.
- [ ] 기록이 존재하지 않는 경우 화면 중앙에 “첫 기록을 추가해보세요” 라벨을 통해 기록 저장을 유도해야 한다.

### 삭제/수정

- [ ] 사용자는 기록을 삭제할 수 있다.
- [ ] 기록 삭제 시 삭제 여부를 묻는 Alert(확인 창)로 재확인한다.
- [ ] 삭제 확인 문구: “정말로 이 항목을 삭제하시겠습니까?”
- [ ] 삭제 확인 알럿에서 취소 시 데이터가 변경되지 않아야 한다.
- [ ] 사용자는 캡션을 수정할 수 있다.
- [ ] 캡션 수정 시 수정된 내용은 주기적으로 서버에 업로드되어야 하며 자동으로 이루어져야 한다.
- [ ] 캡션 수정 중 화면이 닫히는 경우 수정된 내용은 즉시 서버로 전송되어야 한다.
- [ ] 네트워크 오류로 삭제/수정 실패 시 “네트워크 실패로 인해 작업이 실패했습니다” 토스트 메시지가 보여져야 한다.

## 4) 기술 스택 / 구조

- Tuist 기반 모듈 구조
  - `PinItApp` (iOS App): App 엔트리, Presentation/UI, 라우팅
  - `PinItCore` (iOS Static Framework): 도메인 모델, 유스케이스/프로토콜, Validator
  - `PinItTests` (iOS Unit Tests): Validator/도메인 유효성 중심 단위 테스트
- 의존성 방향
  - `PinItApp -> PinItCore`
  - `PinItTests -> PinItCore` (필요 시 `PinItApp` 테스트 가능하도록 타겟 의존 구성)
  - `PinItCore -> PinItApp` 금지
- 주요 기술
  - Swift 6
  - iOS Deployment Target 17.0
  - UIKit + SwiftUI
  - RxSwift / RxCocoa

## 5) 실행 방법

1. Tuist 설치 및 버전 확인

```bash
curl -Ls https://install.tuist.io | bash
tuist version
```

2. 프로젝트 생성

```bash
tuist generate
```

3. Xcode에서 실행

- 생성된 `PinIt.xcworkspace`를 열어 `PinItApp` 스킴을 실행합니다.
